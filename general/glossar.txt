Glossar – Maschinelles Lernen

Action / Aktion
Ein möglicher Zug oder eine Entscheidung, die das Modell auswählen kann. In diesem Projekt entspricht eine Aktion einer (x,z)-Position auf dem Spielfeld.

Arena / Evaluation
Vergleichssystem, in dem ein neues Kandidatenmodell gegen das aktuelle Champion-Modell spielt. Dadurch wird überprüft, ob das neue Modell tatsächlich stärker geworden ist.

Backpropagation
Lernverfahren neuronaler Netzwerke. Nachdem das Modell einen Fehler berechnet hat (Loss), wird dieser Fehler rückwärts durch das Netzwerk propagiert. Dabei werden die internen Gewichte angepasst, damit zukünftige Vorhersagen genauer werden.

Batch Size
Anzahl der Trainingsbeispiele, die gleichzeitig verarbeitet werden. Größere Batches benötigen mehr Speicher, können aber stabileres Training ermöglichen.

Candidate
Ein neu trainiertes Modell, das gegen den aktuellen Champion getestet wird. Nur bei ausreichender Leistung wird es übernommen.

Checkpoint
Gespeicherter Trainingszustand eines Modells. Dadurch kann Training fortgesetzt oder ein älterer Stand wiederhergestellt werden.

Champion
Das aktuell beste und freigegebene Modell des Systems.

Encoder
Komponente, die das Spielfeld in eine Tensorrepräsentation umwandelt, die vom neuronalen Netzwerk verarbeitet werden kann.

Evaluation
Siehe Arena.

Export
Umwandlung eines trainierten Modells in ein Laufzeitformat, damit es außerhalb der Trainingsumgebung genutzt werden kann.

Feature
Ein relevantes Merkmal innerhalb der Eingabedaten. Beispiele sind eigene Steine, gegnerische Steine oder Informationen darüber, welcher Spieler am Zug ist.

Gewichte
Interne numerische Parameter des neuronalen Netzwerks. Sie bestimmen, wie das Modell Informationen verarbeitet und werden während des Trainings angepasst.

Inference
Verwendung eines bereits trainierten Modells zur Berechnung einer Vorhersage oder Entscheidung im laufenden Betrieb.

Input / Eingabe
Die Daten, die an das Modell übergeben werden, zum Beispiel der aktuelle Spielzustand als Tensor.

Learning Rate
Steuert, wie stark die Gewichte bei jedem Trainingsschritt angepasst werden. Zu hohe Werte können Training instabil machen, zu niedrige Werte verlangsamen das Lernen.

Legal Mask
Filter für erlaubte Aktionen. Ungültige Züge werden markiert und bei der Auswahl ausgeschlossen, damit das Modell nur regelkonforme Aktionen ausführen kann.

Loss
Fehlerwert des Modells. Er beschreibt, wie stark die Vorhersage von der gewünschten Zielausgabe abweicht. Ziel des Trainings ist es, diesen Fehler zu minimieren.

Maschinelles Lernen (ML)
Bereich der Informatik, bei dem Systeme aus Beispieldaten lernen, statt alle Regeln manuell programmiert zu bekommen.

MCTS (Monte Carlo Tree Search)
Suchalgorithmus zur Verbesserung der Zugauswahl. Der Algorithmus simuliert mögliche Spielverläufe und bewertet deren Erfolgschancen.

Modell
Die eigentliche KI-Komponente des Systems. Sie verarbeitet Eingaben und berechnet daraus Entscheidungen oder Bewertungen.

Neural Network / CNN / ResNet
Verschiedene Architekturen neuronaler Netzwerke. CNNs erkennen räumliche Muster, während ResNets tiefere Netzwerke mit stabilerem Lernen ermöglichen.

Neuronales Netzwerk
Ein mathematisches Modell aus vielen verbundenen Rechenschichten, das Muster in Daten erkennen und daraus Entscheidungen ableiten kann.

ONNX
Standardisiertes Austauschformat für Machine-Learning-Modelle. Ermöglicht die Nutzung eines trainierten Modells außerhalb von PyTorch.

Output / Ausgabe
Das Ergebnis des Modells, zum Beispiel Zugwahrscheinlichkeiten oder eine Bewertung der aktuellen Spielsituation.

Policy
Teil des Modells, der entscheidet, welche Aktion am sinnvollsten erscheint.

Policy Logits
Rohwerte des Modells für alle möglichen Aktionen. Erst nach weiterer Verarbeitung entstehen daraus Wahrscheinlichkeiten.

Replay Buffer
Speicher für gesammelte Self-Play-Partien. Alte und neue Spiele werden gemischt, damit das Modell aus unterschiedlichen Situationen lernen kann.

Reproduzierbarkeit
Eigenschaft eines Trainingssystems, bei gleichen Einstellungen möglichst identische Ergebnisse zu erzeugen. Dafür werden unter anderem feste Zufallswerte (Seeds) verwendet.

Self-Play
Trainingsverfahren, bei dem das Modell gegen sich selbst spielt, um automatisch neue Trainingsdaten zu erzeugen.

State / Game State
Vollständiger aktueller Zustand des Spiels, inklusive Spielfeld und Information darüber, welcher Spieler am Zug ist.

Tensornale Zahlenstruktur für 
MehrdimensioMachine Learning. Das Spielfeld wird in Tensorform umgewandelt, damit es vom Modell verarbeitet werden kann.

Training
Lernprozess, bei dem das Modell anhand von Beispieldaten verbessert wird.

Training Loop
Wiederholter Ablauf aus Self-Play, Datenspeicherung, Training und Evaluation des Modells.

Trajectory
Kompletter Spielverlauf mit Zuständen, Aktionen und Endergebnis. Diese Daten dienen als Trainingsgrundlage.

Value
Bewertung einer Spielsituation durch das Modell. Werte nahe +1 bedeuten meist Gewinnwahrscheinlichkeit, Werte nahe -1 Verlustwahrscheinlichkeit.

WebSocket
Dauerhafte Netzwerkverbindung zwischen Agent und Spielserver zum Senden und Empfangen von Spielinformationen.

Winrate
Anteil gewonnener Spiele eines Modells. Wird verwendet, um die Stärke verschiedener Modelle zu vergleichen.
